# API 回應格式規範

## 標準格式

### 成功回應

```json
// 單一資源
{ "id": 1, "name": "...", ... }

// 列表
[{ "id": 1, ... }, { "id": 2, ... }]

// 分頁列表
{ "data": [...], "total": 100 }

// 操作結果
{ "message": "操作成功" }
```

### 錯誤回應

```json
{ "error": "錯誤描述" }
```

### HTTP Status Code

| Code | 用途 |
|------|------|
| 200 | 查詢成功、更新成功、刪除成功 |
| 201 | 建立成功 |
| 400 | 請求格式錯誤（缺少必填欄位、格式不對） |
| 401 | 未認證（JWT 無效或過期） |
| 403 | 無權限（角色不符、租戶隔離、配額超限） |
| 404 | 資源不存在 |
| 500 | 伺服器內部錯誤 |

## 現況與遷移計畫

目前部分 API 回應格式不一致（歷史原因），前端 API client 的 interceptor 已處理兼容。
新增 API 一律遵循上述標準，舊 API 逐步遷移。
