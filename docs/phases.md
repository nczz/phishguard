# 階段規劃與驗收條件

## Phase 1 — 核心引擎（你幫客戶操作）

**目標：** 平台管理員可以透過 API 幫客戶建立租戶、執行釣魚測試、查看結果。

**預估時間：** 3 週

### 交付項目

- [x] MySQL schema 建立完成（所有核心表）
- [x] 租戶 CRUD API
- [x] 使用者 CRUD + JWT 認證
- [x] 情境庫 CRUD（模板 + Landing Page + 教育頁）
- [x] 收件人群組 CRUD + CSV 匯入
- [x] SMTP Profile CRUD + 測試發信
- [x] Campaign 建立 + 選人引擎（全選/隨機抽樣/部門篩選）
- [x] Mail Worker 發信（SMTP + Mailgun + SES）
- [x] Track Server（開信/點擊/下載/提交/舉報）
- [x] 漏斗統計 API
- [x] 稽核日誌（自動記錄所有寫入操作）

### 驗收條件

1. **建立租戶** — `POST /api/admin/tenants` 建立租戶，回傳 tenant 資訊
2. **建立使用者** — 為租戶建立管理員帳號，可登入取得 JWT
3. **匯入收件人** — 上傳 CSV，系統正確解析並建立收件人群組
4. **執行 Campaign** — 選擇情境 + 收件人群組 + SMTP，發送成功
5. **追蹤驗證** — 開信/點擊/提交事件正確記錄到 events 表
6. **漏斗統計** — API 回傳正確的 sent/opened/clicked/submitted/reported 數字
7. **租戶隔離** — 租戶 A 無法存取租戶 B 的任何資料
8. **稽核日誌** — 所有 API 寫入操作都有對應的 audit_logs 記錄

## Phase 2 — 客戶自助介面

**目標：** 客戶可以自行登入 Web UI 操作，不需要平台管理員介入。

**預估時間：** 3 週

### 交付項目

- [ ] React 前端：登入頁
- [ ] React 前端：租戶 Dashboard（活動總覽 + 漏斗圖）
- [ ] React 前端：三步驟建立測試（選情境 → 選對象 → 發送）
- [ ] React 前端：Campaign 詳情頁（即時結果 + 時間軸）
- [ ] React 前端：收件人管理（群組 + CSV 匯入）
- [ ] React 前端：報表頁（漏斗 + 部門排名）
- [ ] PDF 報表匯出
- [ ] 報表自動寄送（Campaign 完成 7 天後）
- [ ] 平台管理員 Dashboard（租戶列表 + 用量統計）

### 驗收條件

1. **客戶自助流程** — 客戶登入 → 選情境 → 選對象 → 發送，全程不超過 5 分鐘
2. **即時結果** — Campaign 發送後，Dashboard 即時更新追蹤數據
3. **PDF 報表** — 匯出的 PDF 包含漏斗圖、部門排名、時間軸
4. **平台管理員** — 可查看所有租戶列表、用量、切換到任意租戶操作
5. **權限控制** — Viewer 角色只能看報表，不能建立 Campaign

## Phase 3 — 自動化與差異化

**目標：** 客戶設定一次，系統自動定期執行測試。提供差異化的分析功能。

**預估時間：** 3 週

### 交付項目

- [ ] 自動定期測試（頻率 + 對象 + 情境自動輪替）
- [ ] 分散發送（時間窗口內隨機分佈）
- [ ] 冷卻期（同一人 N 天內不重複測試）
- [ ] 難度分級（情境標記難度 1-3）
- [ ] 累犯追蹤（跨 Campaign 個人歷史）
- [ ] 趨勢分析（季度/年度對比）
- [ ] 自訂模板編輯器（HTML 編輯 + 預覽）
- [ ] 稽核日誌 UI + CSV 匯出

### 驗收條件

1. **自動測試** — 設定每季自動測試，系統在指定時間自動建立 Campaign 並發送
2. **分散發送** — 500 人的 Campaign，信件在 8 小時內隨機分散送出
3. **冷卻期** — 30 天內測過的人不會被再次選中
4. **累犯報表** — 可查看特定員工在過去 N 次測試中的行為紀錄
5. **趨勢圖** — Dashboard 顯示近 4 季的點擊率變化趨勢

## Phase 4 — 企業級功能

**目標：** 滿足中大型企業的採購與合規要求。

**預估時間：** 3 週

### 交付項目

- [ ] SSO 整合（SAML / OIDC）
- [ ] 自訂發信域名（per-tenant）
- [ ] 訂閱管理 + 用量計費
- [ ] 產業對比數據（跨租戶匿名統計）
- [ ] 釣魚信舉報按鈕（Outlook / Gmail 外掛）
- [ ] API Key 認證（供客戶程式化操作）
- [ ] 白標支援（自訂 logo、色彩）

### 驗收條件

1. **SSO** — 客戶可透過 Azure AD 登入，不需另外建帳號
2. **自訂域名** — 租戶設定自訂域名後，追蹤連結使用該域名
3. **計費** — 平台管理員可查看每個租戶的用量與費用
4. **舉報按鈕** — 員工在 Outlook 點擊舉報，事件正確記錄到 Campaign 結果
